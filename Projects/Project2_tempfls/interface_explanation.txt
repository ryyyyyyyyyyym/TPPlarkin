Интерфейс ICommand — задаёт контракт для всех команд в игре.
Содержит свойство Name (свойство, хранит имя команды) и метод Execute (метод, выполняет команду и принимает состояние игры и аргументы пользователя).

Интерфейс IInteractable — задаёт контракт для объектов взаимодействия. 
Содержит свойство Id (свойство, уникальный идентификатор объекта) и метод Interact (метод, описывает взаимодействие с объектом).

Интерфейс ICondition — интерфейс условий. 
Содержит метод Check (метод, возвращает bool и проверяет состояние игры).

Интерфейс IEffect — интерфейс эффектов. 
Содержит метод Apply (метод, изменяет состояние игры).

Абстрактный класс CommandBase — базовый класс для команд, реализует интерфейс ICommand. 
Содержит свойство Name (свойство, имя команды, protected set — можно изменять только внутри класса и наследников), конструктор CommandBase (конструктор, задаёт имя команды) и абстрактный метод Execute (метод, должен быть реализован в наследниках).

Абстрактный класс ConditionBase — базовый класс условий, реализует интерфейс ICondition. 
Содержит абстрактный метод Check (метод, должен быть реализован в конкретных условиях).

Абстрактный класс EffectBase — базовый класс эффектов, реализует интерфейс IEffect. 
Содержит абстрактный метод Apply (метод, должен быть реализован в конкретных эффектах).

Абстрактный класс GameEventBase — базовый класс событий. 
Содержит поле condition (поле, хранит условие срабатывания события) и поле effects (поле, массив эффектов). Содержит конструктор GameEventBase (конструктор, принимает условие и массив эффектов и сохраняет их в поля класса). Содержит метод Execute (метод, проверяет условие через condition.Check(state), и если условие выполняется, то с помощью цикла for последовательно вызывает метод Apply у каждого эффекта из массива effects).
