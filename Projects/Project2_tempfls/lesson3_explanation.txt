Урок 3. Реализация конкретных классов

На данном этапе были реализованы конкретные классы условий, эффектов и объектов взаимодействия. Эти элементы формируют основную игровую логику и позволяют гибко управлять поведением игры через композицию объектов.

Условия (Conditions)

Условия используются для проверки состояния игры. Каждый класс реализует метод Check(GameState state), который возвращает true или false.

HasItemCondition — проверяет, есть ли у игрока определённый предмет.
FlagCondition — проверяет значение флага (например, включён ли генератор).
HealthCondition — проверяет уровень здоровья игрока.
AndCondition — возвращает true, если оба условия выполняются.
OrCondition — возвращает true, если выполняется хотя бы одно условие.
NotCondition — инвертирует результат условия.

Таким образом можно строить сложные логические проверки, комбинируя простые условия.

Эффекты (Effects)

Эффекты изменяют состояние игры. Каждый класс реализует метод Apply(GameState state).

AddItemEffect — добавляет предмет в инвентарь.
RemoveItemEffect — удаляет предмет из инвентаря.
SetFlagEffect — устанавливает или изменяет флаг.
DamageEffect — наносит урон игроку.
HealEffect — восстанавливает здоровье.
LogEffect — записывает сообщение в журнал событий.

Эффекты применяются последовательно и позволяют реализовать любые изменения состояния игры.

Объекты взаимодействия (Interactables)

Объекты реализуют интерфейс IInteractable и содержат метод Interact(GameState state), который вызывается при взаимодействии игрока.

Chest — контейнер, который можно открыть один раз. После открытия применяет эффекты (например, выдаёт предмет).
Door — дверь, которая открывается только при выполнении условия.
NPC — персонаж или терминал, реагирующий на состояние игры.
Trap — ловушка, срабатывающая один раз и применяющая эффекты (например, урон).

Каждый объект самостоятельно определяет свою логику поведения, что соответствует принципу инкапсуляции.

Локации (Location)

Класс Location описывает игровую локацию и хранит:

название (Name)
описание (Description)
объекты (Objects)
переходы (Exits)
события (Events)

Методы:

FindObject(string id) — ищет объект по идентификатору.
TriggerEvents(GameState state) — запускает все события локации.
Describe() — выводит описание локации, объектов и доступных выходов.

Локация является основным элементом игрового мира и объединяет все его компоненты.

Общий принцип работы

Игровая логика строится через взаимодействие трёх компонентов:

Условия проверяют состояние игры
Эффекты изменяют состояние
События и объекты объединяют условия и эффекты

Это позволяет создавать гибкую архитектуру без жёсткой привязки к конкретным классам и избегать проверки типов объектов.
